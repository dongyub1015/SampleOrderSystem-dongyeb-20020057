# PRD: 반도체 시료 생산주문관리 시스템

## 1. 개요

### 1.1 배경
가상의 반도체 회사 **S Semi**는 연구소, 팹리스(Fabless) 업체, 대학 연구실 등의 고객에게 반도체 시료를 생산·납품하고 있다. 주문량 급증으로 인해 엑셀·메모장 기반의 기존 관리 방식에서 실수가 잦아지고, 재고 및 공정 현황 파악이 어려워졌다. 이를 해결하기 위해 콘솔 기반의 생산주문관리 시스템을 개발한다.

### 1.2 목표
- 시료 등록 및 주문 접수부터 출고까지의 전 과정을 일관된 시스템으로 관리
- 재고 및 생산 현황을 실시간으로 모니터링
- 담당자가 콘솔 명령을 통해 직접 운영

### 1.3 범위
- 콘솔(CLI) 기반 단일 프로세스 애플리케이션
- 시료 관리, 주문 처리, 생산 라인, 모니터링, 출고 처리 기능 포함

---

## 2. 사용자

| 역할 | 설명 |
|------|------|
| 주문 담당자 | 고객 주문 접수 및 승인/거절 처리 |
| 생산 담당자 | 시료 등록, 생산 라인 현황 확인, 출고 처리 |

---

## 3. 핵심 도메인

### 3.1 시료 (Sample)
시스템의 가장 기본이 되는 단위. 시스템에 등록된 시료만 주문 가능.

| 속성 | 설명 |
|------|------|
| 시료 ID | 고유 식별자 |
| 이름 | 시료 명칭 |
| 평균 생산시간 | 시료 1개 생산에 소요되는 평균 시간 (min/ea) |
| 수율 | 정상 시료 수 / 총 생산 시료 수 (예: 0.92) |
| 현재 재고 | 현재 보유 중인 정상 시료 수량 |

### 3.2 주문 (Order)
모든 주문은 아래 상태 중 하나를 가진다.

| 상태 | 의미 |
|------|------|
| RESERVED | 주문 접수 완료 |
| REJECTED | 주문 거절 (비정상 흐름, 모니터링 제외) |
| PRODUCING | 주문 승인 완료 + 재고 부족으로 생산 중 |
| CONFIRMED | 주문 승인 완료 + 출고 대기 중 |
| RELEASE | 출고 완료 |

**상태 전이 흐름:**
```
RESERVED ──[승인, 재고 충분]──> CONFIRMED ──[출고]──> RELEASE
RESERVED ──[승인, 재고 부족]──> PRODUCING ──[생산 완료]──> CONFIRMED
RESERVED ──[거절]──> REJECTED
```

### 3.3 생산 라인 (Production Line)
- 시료를 하나씩 생산하는 설비 단위
- 주문이 들어온 시료에 대해서만 생산 수행
- 생산 큐(Queue)는 **FIFO** 방식으로 스케줄링

## 4. 기능 요구사항

### 4.1 메인 메뉴
- 기능별 선택 화면 표시
- 전체 시료에 대한 요약 정보(등록 수, 총 재고, 진행 중 주문 수) 표시

```
반도체 시료 생산주문관리 시스템

시스템 현황: YYYY-MM-DD HH:MM:SS
등록 시료 N개  총 재고 N ea  진행 주문 N건  생산대기 N건(예기)

[1] 시료 관리     [2] 시료 주문
[3] 주문 승인/거절  [4] 모니터링
[5] 생산라인 조회  [6] 출고 처리
[0] 종료

선택 > _
```

---

### 4.2 시료 관리

#### 4.2.1 시료 등록
- 새로운 시료를 시스템에 추가
- 입력 항목: 시료 ID, 이름, 평균 생산시간, 수율
- 중복 시료 ID 등록 불가

#### 4.2.2 시료 목록 조회
- 등록된 모든 시료 목록 출력
- 표시 항목: 시료 ID, 이름, 평균 생산시간, 수율, 현재 재고 수량

#### 4.2.3 시료 검색
- 이름 등 속성 기반으로 특정 시료 검색

---

### 4.3 시료 주문

#### 4.3.1 주문 접수 (예약)
- 고객이 원하는 시료와 수량을 주문
- 주문 생성 시 상태: **RESERVED**
- 입력 항목: 시료 ID, 고객명, 주문 수량
- 주문번호 자동 생성 (예: `ORD-YYYYMMDD-NNNN`)

#### 4.3.2 주문 승인
- RESERVED 상태의 주문 목록을 조회 후 특정 주문 승인
- 승인 시 재고 상황에 따라 자동 분기 처리:
  - **재고 충분** → 즉시 `CONFIRMED` 전환
  - **재고 부족** → 생산 라인 큐에 자동 등록, `PRODUCING` 전환

#### 4.3.3 주문 거절
- RESERVED 상태의 특정 주문 거절
- 즉시 `REJECTED` 상태로 전환

---

### 4.4 모니터링

#### 4.4.1 주문량 확인
- 현재 상태별 주문 현황 표시: `RESERVED`, `CONFIRMED`, `PRODUCING`, `RELEASE`
- `REJECTED`는 표시 제외

#### 4.4.2 재고량 확인
- 각 시료별 현재 재고 수량 표시
- 주문 대비 재고 상태 표기:

| 상태 | 조건 |
|------|------|
| 여유 | 주문 대비 재고 충분 |
| 부족 | 주문 대비 재고 수량 부족 |
| 고갈 | 재고 수량 = 0 |

---

### 4.5 생산 라인

#### 4.5.1 생산 수량 계산
```
실 생산량 = ceil(부족분 / (수율 * 0.9))
총 생산 시간 = 평균 생산시간 * 실 생산량
```

#### 4.5.2 생산 현황 표기
- 현재 생산 중인 시료 정보 출력
- 표시 항목: 주문 정보, 재고 → 부족분, 실 생산량, 예상 완료 시각 등 (수준 자율 결정)

#### 4.5.3 대기 주문 확인
- 생산 큐(FIFO)에 대기 중인 작업 목록 출력
- 표시 항목: 순서, 주문번호, 시료명, 주문량, 부족분, 실 생산량, 예상 완료 시각

#### 4.5.4 생산 완료 처리
- 생산 완료 시 해당 주문 상태: `PRODUCING` → `CONFIRMED` 자동 전환
- 생산된 시료 재고에 반영

---

### 4.6 출고 처리
- `CONFIRMED` 상태의 주문 목록 표시
- 특정 주문에 대해 출고 실행
- 출고 완료 시 주문 상태: `CONFIRMED` → `RELEASE` 전환
- 출고 수량만큼 재고에서 차감
- 표시 항목: 주문번호, 출고 수량, 처리 일시, 상태 변경 결과

---

## 5. 비기능 요구사항

| 항목 | 내용 |
|------|------|
| 인터페이스 | 콘솔(CLI) 기반, 담당자가 직접 명령 입력 |
| 데이터 저장 | 프로세스 실행 중 인메모리 유지 (영속성 요구사항 별도 정의 없음) |
| 스케줄링 | 생산 큐는 FIFO 방식 |
| 언어 | 제약 없음 (구현 언어 자율 선택) |

---

## 6. 주문번호 규칙
- 형식: `ORD-YYYYMMDD-NNNN` (예: `ORD-20260508-0001`)
- 날짜 기반 일련번호 자동 채번

---

## 7. 용어 정의

| 용어 | 정의 |
|------|------|
| 시료 (Sample) | S Semi가 생산·납품하는 반도체 제품 단위 |
| 수율 (Yield) | 정상 시료 수 / 총 생산 시료 수 |
| 생산 라인 | 시료를 하나씩 생산하는 설비 흐름 단위 |
| 부족분 | 주문 수량 - 현재 재고 수량 (음수이면 0으로 처리) |
| 실 생산량 | 수율과 오차를 고려한 실제 생산 투입량 |
