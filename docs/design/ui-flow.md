# UI 흐름 설계 (Console UI Flow)

## 1. 전체 메뉴 계층

```
메인 메뉴
├── [1] 시료 관리
│     ├── [1] 시료 등록
│     ├── [2] 시료 목록 조회
│     ├── [3] 시료 검색
│     └── [0] 뒤로
├── [2] 시료 주문
│     ├── [1] 주문 접수 (예약)
│     ├── [2] 주문 승인/거절
│     └── [0] 뒤로
├── [3] (구 메뉴 → [2]로 통합)
├── [4] 모니터링
│     ├── [1] 주문별 현황
│     ├── [2] 재고량 확인
│     └── [0] 뒤로
├── [5] 생산 라인 조회
│     └── (자동 표시 후 뒤로)
├── [6] 출고 처리
│     └── (CONFIRMED 목록 표시 → 출고 번호 입력)
└── [0] 종료
```

## 3. 공통 UX 규칙

| 규칙 | 내용 |
|------|------|
| 구분선 | `═` (이중선): 화면 제목 / `─` (단선): 목록 구분 |
| 입력 프롬프트 | `선택 > _` 형식 통일 |
| 성공 메시지 | `✔` 접두사 사용 |
| 취소/뒤로 | `[0]` 또는 `[N]` 키로 일관성 유지 |
| 빈 목록 | "조회 결과가 없습니다." 메시지 출력 |
| 잘못된 입력 | "올바른 번호를 입력해 주세요." 후 재입력 요청 |
| 페이지네이션 | 목록이 20건 초과 시 `[N] 다음` / `[P] 이전` 제공 |
